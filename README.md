# NetPractice

## Description

NetPractice is a practical networking exercise designed to develop a solid understanding of **TCP/IP addressing** and **network configuration**. The goal is to solve 10 progressively challenging levels, each presenting a broken or incomplete network diagram that must be repaired so that all devices can communicate correctly.

Through this project, you will learn how IP addresses, subnet masks, routing tables, and gateways interact to form a functional network. By the end, you should be comfortable reading and configuring any small-scale network topology.

### What you will work with

- **IP addressing** — understanding IPv4 notation, address classes, and how hosts are identified on a network
- **Subnet masks** — calculating valid host ranges, network addresses, and broadcast addresses using CIDR notation (e.g. `/24`, `/28`) and dotted-decimal masks
- **Default gateways** — routing traffic from a local network to external destinations
- **Routers and switches** — understanding how routers separate networks and how switches connect devices within the same network
- **Routing tables** — configuring static routes so that packets know where to go when leaving a subnet
- **OSI model** — understanding at which layers (Network layer — Layer 3, Data Link — Layer 2) the concepts in this project operate
- **Subnetting** — dividing address spaces to accommodate multiple networks without conflicts

## Instructions

### Running the training interface

A browser-based training interface is provided. To launch it:

```bash
# Clone the repository (if you haven't already)
git clone <your_repo_url>
cd netpractice

# Launch the interface
bash run.sh
```

This will open the NetPractice interface in your default browser. From there, you can navigate between the 10 levels and test your configurations in real time.

> If `run.sh` is not available in your environment, open `index.html` directly in your browser.

### Solving levels

Each level presents a network diagram with some fields left empty or misconfigured. Your task is to fill in the missing values (IP addresses, masks, gateway addresses, or route destinations/next hops) so that all devices on the diagram can reach each other.

Use the **"Check"** button to validate your solution. Once a level passes, export the configuration before moving on.

### Exporting configurations

Once you have solved a level, export the configuration using the **export button** in the interface. This will download a `.json` file for that level.

Repeat this for all 10 levels.

### Submission requirements

> **10 exported configuration files (one per level) must be placed at the root of the repository.**

The files should follow this naming convention (as generated by the interface):

```
level1.json
level2.json
...
level10.json
```

Make sure all 10 files are present at the root before pushing:

```bash
ls *.json        # should list 10 files
git add *.json
git commit -m "add netpractice level configs"
git push
```

### Concepts specifically studied

- **TCP/IP addressing** — IPv4 address structure, network vs. host portions
- **Subnet masks** — dotted-decimal and CIDR notation, valid ranges, network/broadcast addresses
- **Default gateways** — routing traffic outside the local subnet
- **Routers** — separating networks, forwarding packets based on routing tables
- **Switches** — connecting devices within the same network at Layer 2
- **Static routing** — manually defined routes with destination network and next-hop
- **OSI layers** — focus on Layer 3 (Network) for IP routing and Layer 2 (Data Link) for switching
